## v0.6.0 — Production-grade anonymizace (~16:00)

**Reakce na Michalovo review** demo PDF z Jiříkova spisu — identifikoval 4 kategorie problémů upstream MasKITu:

### Diagnostika problémů
1. 🔴 **MasKIT halucinace** na běžných slovech:
   - `"státu" → "UniAgentury"`
   - `"sporu" → "Pardubic"`
   - `"materiální" → "Zlín"`
   - `"obyvatel" → "Pavla"`
2. 🔴 **Telefon fragmentace**: `"777 18 18 10" → "123 18 18 10"` (jen první 3 cifry nahrazeny)
3. 🔴 **Adresy fragmented**: `"Opavě 01 Opava" → "Praze"`
4. 🔴 **Random fake names** = nereprodukovatelné (Jiří → "Jan" dnes, "Petr" zítra)

### Co bylo implementováno — 8-krokový pipeline v `maskit.py`

1. **Regex pre-pass** (`regex_pre_pass=True` default):
   - Format patterns (no false positives): e-mail, URL, telefon CZ/SK (3 formáty: 3-3-3, 3-2-2-2, +420 prefix), RČ, DIČ, SPZ, IBAN
   - Context patterns (vyžadují prefix): IČO:, PSČ:, č.j., sp. zn. (+ "spisová značka:"), občanský průkaz, datová schránka
   - **Court regex**: chytá celé jméno soudu vč. lokality (Krajský soud v Ostravě, Mestský súd Bratislava II, Ústavní soud České republiky, Najvyšší súd SR)

2. **Strict pre-pass** (`strict=True`): NameTag firmy/úřady/instituce → FIRMA1/INSTITUCE1 sentinely

3. **MasKIT volání** na text se sentinely

4. **Stop-list filter** (`stop_list_filter=True`):
   - 50+ známých CZ slov co MasKIT halucinuje (stát, republika, spor, materiální, obyvatel, vláda, úřad, měsíce, právní termíny)
   - Post-process rollback: pokud MasKIT nahradil → vrátit originál + warning
   - **Test: chytil 4/4 viditelné halucinace** v Jiříkově spisu

5. **Restore sentinely** → finální placeholdery (TELEFON1, FIRMA1, ICO1, …)

6. **Fragmentation detection warnings**

7. **Type classification** — NameTag čtyřvrstvý fallback

8. **Placeholder mode** (`placeholder_mode=True` opt-in):
   - Místo MasKIT random fake names → deterministické OSOBA1, MESTO1, ULICE1, INSTITUCE1
   - Konzistentní dedup: "Tóthová" × 3 v textu → vždy stejný placeholder
   - Reprodukovatelné (audit/peer review), auditovatelné (1:1 mapping)
   - 17 typů prefixů: OSOBA, ULICE, MESTO, STAT, REGION, FIRMA, INSTITUCE, EMAIL, TELEFON, ICO, PSC, RC, CJ, SPZN, OP, DATOVKA, IBAN

### Sentinel saga
Cesta k funkční implementaci PUA sentinelů (předchozí pokusy selhaly):
- ❌ `__PIIPRE0001__` — MasKIT zpracoval "PRE" jako prefix
- ❌ `ZxZP0001PZxZ` — velká písmena tokenizovaná
- ❌ `§§§p0001p§§§` — § rozkládá tokenizer
- ❌ `xqxqxq0001xqxqxq` — MasKIT detekoval jako kód/email
- ❌ PUA chars + digit ID — digits uvnitř tokenizovány
- ✅ **Single PUA character** (U+E100-E2FF) — MasKIT slovník neobsahuje

### Rebuild anonymized v placeholder mode
Předchozí approach (`string.replace` MasKIT_plc → new_plc) měl bug: krátký MasKIT placeholder "B" by nahradil "B" v "Bydliště" → "OSOBA11ydliště". Refaktor: walk raw + `_MASKIT_PLACEHOLDER` regex finditer + per-position substitution.

### Řešení — `langdetect.py` (NEW)

Sdílený modul pro NameTag i UDPipe s 3-vrstvou strategií:

**1. Non-Latin skripty** (immediate return):
- Cyrilice s rozlišením UK chars (`іїєґ`) → ukrainian, jinak → russian
- CJK, hiragana/katakana → chinese/japanese rozlišené
- Hangul → korean
- Arabic, Hebrew, Devanagari, Thai, Greek

**2. Character signatures** (distinktivní diakritika):
- Vietnamština: `ă/â/đ/ê/ô/ơ/ư`
- Estonština: `õ` (unique vs SE/DE)
- Rumunština: `ț/ș`
- Skandinávské: `æ/ø/å` (pak rozliš DA/NO/SV podle nejvyššího markeru skóre)
- Turečtina: `ı/ş/ğ` + dotless `İ`

**3. Score-based** pro 21 latinkových jazyků (SK/HU/FI/LT/LV/PL/RO/SL/HR/SR/NL/DE/PT/ES/IT/FR/DA/SV/NO/EN/CZ):
- Markery pro každý jazyk s vlastním thresholdem
- **CZ proxy**: používá **jen `ř/ě/ů`** (unique pro CZ, NE `š/č/ž` které sdílí SK/SL/HR)
- Score = počet matches; lang s nejvyšším score nad thresholdem vyhrává

### Pattern improvements
- IT/PT/ES/EN/FR: odebrány single-char markery (`i`, `e`, `o`, `a`) které false-positively chytaly cizí texty
- ET: snížen threshold 3→2 + Tallinn/Tartu/Eesti markery
- FI: rozšířena agglutinative suffixy `\w{3,}(ssa|ssä|sta|stä|lla|llä|iin)`
- SK: rozšířeno (mestský, krajský, okresný, prejednáva, návrh, otcovi)

## První reálný PDF test (~20:40)

Michal poslal **iScanner sken** `Child Support Statement - by iScanner.pdf` (1 A4, SK text — vyjadrenie matky Andrei Tóthové → Mestský súd Bratislava II).

### Findings (3 bugs + 1 architekturní insight)

**Bug 1: Vietnamština false positive na SK textech**
- `_VIETNAMESE_CHARS` obsahoval `ô` (U+00F4) jako VI signál
- ALE `ô` se používá v slovenštině (`môj`/`môže`/`môjho`)
- Fix: jen jednoznačné VI chars (`ư/ơ/đ` + složeniny `ễ/ử/ự/ợ/ờ/ằ/ặ/ề`)

**Bug 2: Slovenian false positive na SK textech**
- `_SLOVENIAN_HINT` měl hard-coded early return s `je|so|si|ki` = common Slavic
- SK text "moja dcéra" + "som" + "si" matchovalo SL hint → "slovenian"
- Fix: odebrán early return, SL pattern přepsán na distinktivní (`jaz/moj/Ljubljan/slovensk`)

**Bug 3: HR/SR/LT/PT false positives na common Slavic words**
- HR měl `je|sa|na|za` = velmi common, generoval 50+ score na SK textech
- Fix: všechny patterny přepsány na distinktivní (Zagreb/hrvatsk, Beograd/srpsk, Vilni/lietuv)

**Architekturní insight**: **Pro CZ-mutual-intelligible jazyky (slovak) je CZ CNEC LEPŠÍ než multilingvální UNER**

### Lessons learned

1. **Real-world stress test > syntetické testy**
   - Za 30 minut na 2 PDF (1 print + 1 rukopis) jsme našli 4 bugy
   - Žádný `test_5_cases.py` ani `test_jirik_full.py` (s syntetickými texty) to nezachytil
   - Tj. **Jiříkův víkendový test je nesmírně cenný**

2. **MasKIT má limity** — trained jen na úřední/právní text
   - Emocionální texty (rukopis, chat, messengery) → 0 replacements
   - **`placeholder_mode=True` je nyní doporučený default** pro production legal
   - NameTag fallback automatically chytá co MasKIT vynechá

3. **CZ-mutual-intelligible jazyky → použij CZ CNEC**
   - SK ne přes multilingvální UNER, ale přes CNEC 2.0
   - 6× víc entit (24 vs 4) díky bohatému CZ tagsetu

4. **Architecture matters** — disciplína 400 LOC/soubor
   - 18 modulů, žádný přes 400 = navigovatelné, refaktorovatelné
   - PR review-friendly

5. **Builder flow funguje** — Michalovo *"baví mě to a dává smysl"*
   - Krátký feedback loop (review → fix → release → další)
   - Žádné porady, žádné "plánování pro Q3"
   - Quality > speed paradox vyřešen: kvalita PŘICHÁZÍ z rychlé iterace s reálnými testy
